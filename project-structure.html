⚙️ 설정 가이드

🚨 긴급! 다음 설정들이 완료되지 않아 자동 배포가 작동하지 않습니다.

1단계: GitHub Secrets 설정 (최우선)

  1. GitHub Secrets 페이지로 이동
  2. "New repository secret" 클릭
  3. FIREBASE_TOKEN 설정:
    1//0gAPQL2pA35NPCgYIARAAGBASNwF-L9IrfFhKm0PxCSgF8CUhBnddPfW-KFndg5s5l6YUBo3WpzamLHg-1pRL269_ExnDhYIxHfg
  4. FIREBASE_SERVICE_ACCOUNT 설정: docs/current-firebase-tokens.md 파일의 JSON 전체 내용

2단계: Firebase 클라이언트 설정

  1. Firebase Console로 이동
  2. "Your apps" 섹션에서 웹 앱 설정 확인
  3. 다음 값들을 GitHub Secrets에 추가:
    NEXT_PUBLIC_FIREBASE_API_KEY
    NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN (값: wiz-coco.firebaseapp.com)
    NEXT_PUBLIC_FIREBASE_PROJECT_ID (값: wiz-coco)
    NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET (값: wiz-coco.appspot.com)
    NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID
    NEXT_PUBLIC_FIREBASE_APP_ID
    NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID

3단계: 로컬 환경 변수 설정

  1. 프로젝트 루트에 .env.local 파일 생성
  2. env.local.example 파일을 복사하여 실제 값으로 수정
  3. Firebase Console에서 가져온 실제 값들로 교체

4단계: 자동 배포 테스트

  1. 모든 설정 완료 후 main 브랜치에 푸시
  2. GitHub Actions에서 배포 상태 확인
  3. 배포 완료 후 메인 사이트에서 확인
✅ 완료 시 예상 결과: main 브랜치에 코드를 푸시하면 자동으로 빌드 및 배포가 실행되어 https://wiz-coco.web.app 에 반영됩니다.
